Dissertation Project - Preserving Video Game History: Gameboy Emulator

A Gameboy emulator was created from the ground up using C++ with the SDL2 graphics library. The goal for 'good enough' playability, with only minimal impact to gameplay performance allowed. Sound emulation was ruled out from the beginning due to its complexity. The end product is able to run the vast majority of original Gameboy games, with only very minor graphical errors on a small number of games. Agile techniques (MoSCoW, Timeboxing and Kanban) were used to ensure features with the highest benefit to the end product were developed first and any issues were identified and delt with before they could have any drastic consequences to the development process.

Challenges:
